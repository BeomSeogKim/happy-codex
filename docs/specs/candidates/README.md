# Candidate Specs

Create new exploratory or candidate spec documents in this directory.

Use this folder for:

- early concept specs
- alternate gameplay variants
- agent-generated proposal specs
- experiments that are not yet selected for implementation
- candidates that have not been promoted into the top-three comparison

When a candidate becomes selected or ranked, move or copy the durable spec to
`docs/specs/` and update:

- `docs/specs/top-three-candidates.md`
- `README.md`
- `docs/project-setup.md`
- `.codex/skills/day-off-design/SKILL.md`, if workflow guidance changes

Suggested filename format:

```text
kebab-case-concept-name.md
```

Example:

```text
quiet-console-runway.md
```
